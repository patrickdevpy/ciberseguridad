Controles

Mientras que los frameworks se utilizan para crear planes para hacer frente a los riesgos de Seguridad, amenazas y vulnerabilidades , 
los controles se utilizan para reducir riesgos específicos. Si no se aplican los controles adecuados , una organización podría enfrentarse a 
importantes impactos financieros y a daños en su Reputación debido a la exposición a riesgos incluyendo el allanamiento de morada, la 
creación de cuentas falsas de empleados, o la concesión de beneficios gratuitos.

Revisemos la definición de controles. Los controles de seguridad son salvaguardas diseñadas para reducir riesgos de seguridad específicos . 
En este vídeo, hablaremos de tres tipos comunes de controles: encriptación, autenticación y autorización. 
La encriptación es el proceso de convertir datos de un formato legible a un formato codificado. Típicamente, la encriptación implica convertir 
datos de texto plano a texto cifrado. El texto cifrado es el mensaje en bruto, codificado que es ilegible para humanos y ordenadores. 
Los datos de texto cifrado no pueden leerse hasta que se hayan desencriptado a su forma original de texto plano. 
La encriptación se utiliza para garantizar la confidencialidad de los datos sensibles , como la información de las cuentas de los clientes o 
los números de la seguridad social.

Otro control que puede utilizarse para proteger datos confidenciales es la autenticación. La autenticación es el proceso de verificar quién 
es alguien o algo. Un ejemplo real de autenticación es registrarse en un sitio web con su nombre de usuario y contraseña. E
sta forma básica de autenticación demuestra que usted conoce el nombre de usuario y la contraseña y que se le debe permitir el acceso al s
itio web. Métodos de autenticación más avanzados , como la autenticación multifactor, o MFA , desafían al usuario a demostrar que es quien 
dice ser exigiendo tanto una contraseña como una forma adicional de autenticación , como un código de seguridad o datos biométricos , 
como una huella dactilar , una voz o un escáner facial. Los datos biométricos son características físicas únicas que pueden utilizarse para 
verificar la identidad de una persona. Ejemplos de datos biométricos son una huella dactilar , un escáner ocular o un escáner de la palma de 
la mano. Un ejemplo de ataque de ingeniería social que puede explotar los datos biométricos es el vishing . 
El vishing es la explotación de comunicaciones electrónicas de voz para obtener información sensible o suplantar la identidad de una fuente 
conocida. Por ejemplo, el vishing podría utilizarse para suplantar la voz de una persona para robar su identidad y luego cometer un delito. 

Otro control de seguridad muy importante es la autorización. La autorización se refiere al concepto de otorgar acceso a recursos específicos 
dentro de un sistema. Esencialmente, la autorización se utiliza para verificar que una persona tiene permiso para acceder a un recurso . 
Por ejemplo, si trabaja como analista de seguridad de nivel básico para el gobierno federal , podría tener permiso para acceder a datos a 
través de la deep web o otros datos internos a los que sólo puede acceder si es empleado federal. 
Los Controles de seguridad de los que hemos hablado hoy son sólo un elemento de un modelo de seguridad básico conocido como la tríada CID.
