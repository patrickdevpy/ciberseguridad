GUI frente a CLI

Ahora que ha aprendido el funcionamiento interno de las computadoras , analicemos cómo los usuarios y los sistemas operativos se comunican 
entre sí. Hasta ahora, ha aprendido que una computadora tiene un sistema operativo, hardware y aplicaciones. 
Recuerde que el sistema operativo se comunica con el hardware para ejecutar las tareas. 
En este vídeo, aprenderá cómo el usuario( es decir, usted) interactúa con el sistema operativo para enviar tareas al hardware. 
El usuario se comunica con el sistema operativo a través de una interfaz. Una interfaz de usuario es un programa que permite al usuario controlar las funciones del sistema operativo. Las dos interfaces de usuario que analizaremos son la interfaz gráfica de usuario, o GUI, y la interfaz de línea de comandos, o CLI. Vamos a tratar estas interfaces con más detalle.

Una GUI es una interfaz de usuario que usa íconos en la pantalla para administrar diferentes tareas en la computadora. 
La mayoría de los sistemas operativos se pueden usar con una interfaz gráfica de usuario. 
Si ha utilizado una computadora personal o un teléfono celular , tiene experiencia en el manejo de una GUI. 
La mayoría de las GUI incluyen estos componentes : 
un menú de inicio con grupos de programas , una barra de tareas para iniciar programas y un escritorio con iconos y accesos directos. 
Todos estos componentes ayudan a comunicarse con el sistema operativo para ejecutar las tareas. 
Además de hacer clic en los iconos , cuando utiliza una GUI , también puede buscar archivos o aplicaciones desde el menú de inicio. 
Solo tienes que recordar el icono o el nombre del programa para activar una aplicación.

Ahora analicemos la interfaz de línea de comandos. 
En comparación, la interfaz de línea de comandos, o CLI es una interfaz de usuario basada en texto que usa comandos para interactuar con 
la computadora. 
Estos comandos se comunican con el sistema operativo y ejecutan tareas como abrir programas. 
La interfaz de línea de comandos es una estructura muy diferente a la de la interfaz gráfica de usuario. 
Cuando utilice la CLI , notará inmediatamente la diferencia. 
No hay iconos ni gráficos en la pantalla. La interfaz de línea de comandos tiene un aspecto similar al de las líneas de código que 
utilizan ciertos lenguajes de texto. 
Una CLI es más flexible y potente que una GUI. 
Piense en usar una CLI , como crear cualquier comida que desee a partir de ingredientes comprados en una tienda de abarrotes. 
Esto te da mucho control y personalización sobre lo que vas a comer.

En comparación, usar una GUI es más parecido a pedir comida en un restaurante. 
Solo puedes pedir lo que hay en el menú. 
Si quieres tanto un plato de fideos como una pizza , pero el primer restaurante al que vayas solo tiene pizza , tendrás que ir a otro 
restaurante para pedir los fideos. 
Con una interfaz gráfica de usuario , debe realizar una tarea a la vez. 
Sin embargo, la interfaz de línea de comandos permite la personalización, lo que le permite completar varias tareas simultáneamente. 
Por ejemplo, imagine que tiene una carpeta con cientos de archivos de diferentes tipos de archivos y solo necesita mover 
los archivos JPEG a una nueva carpeta. 
Piense en lo lento y tedioso que sería usar una GUI para encontrar cada archivo JPEG de esta carpeta y moverlo a la nueva. 
Por otro lado, la CLI le permitiría agilizar este proceso y moverlos todos a la vez.

Como puede ver, hay diferencias muy grandes en estos dos tipos de interfaces de usuario. 
Como analista de seguridad , parte de su trabajo puede incluir la interfaz de línea de comandos. Al analizar los registros o autenticar 
y autorizar a los usuarios, los analistas de Seguridad suelen utilizar una CLI en su trabajo diario. 
En este vídeo, analizamos dos tipos de interfaces de usuario. 
Descubrió que ya tiene experiencia en el uso de una interfaz gráfica de usuario , ya que la mayoría de los ordenadores personales y 
teléfonos móviles utilizan una GUI. 
Se le presentó la interfaz de línea de comandos.
